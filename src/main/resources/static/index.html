Java 21 VPS Deploy

Procedimento oficial para criar aplicações Java com Docker Multi-Stage Build, clonar do GitHub e expor via Hostinger.

1. Arquivo Dockerfile (No Windows)

Crie um arquivo chamado Dockerfile (sem extensão) na raiz do seu projeto. Ele usa Maven para compilar e JRE para rodar.

# Estágio 1: Build (Maven + JDK 21)
FROM maven:3.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline
COPY src ./src
RUN mvn clean package -DskipTests

# Estágio 2: Run (Apenas JRE Leve)
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

2. Subir código (GitHub)

No terminal do seu VS Code, envie as alterações para nuvem.

git add .

git commit -m "Config Dockerfile Java"

git push origin master

3. Configurar VPS (Clone & Build)

Passo A: Clonar o Projeto

cd /root/infra-proxy git clone https://github.com/srodrigo28/java-vps.git

Passo B: Editar Docker Compose

Edite o arquivo nano docker-compose.yml e adicione o serviço:

  # --- SEU APP JAVA (Repo: java-vps) ---
  app-java:
    build: ./java-vps
    container_name: java-producao
    restart: always
    environment:
      - SPRING_PROFILES_ACTIVE=prod
    networks:
      - rede_interna

Passo C: Subir e Compilar

docker compose up -d --build

4. Inauguração (DNS & Nginx)

A. Hostinger (Zona DNS)

Crie o registro para a internet encontrar sua VPS.

B. Nginx + SSL

Adicione ao nano conf/default.conf:

server {
    listen 80;
    server_name java.99dev.pro;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        proxy_pass http://app-java:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

1. Reiniciar o Porteiro:

docker restart porteiro-nginx

2. Gerar Cadeado (Após DNS propagar):

docker run ... (comando certbot) ... -d java.99dev.pro ...

Ciclo de Atualização

Atualizou o código no Windows?

Siga esta sequência exata na VPS para baixar o novo código e recompilar o Java automaticamente:

cd /root/infra-proxy/java-vps

# 1. Baixa do GitHub
git pull

# 2. Volta para raiz da infra
cd ..

# 3. Recria SÓ o container Java (Build automático)
docker compose up -d --build app-java
